#### #34 R2 バックアップ実装の詳細
- `scripts/r2-backup.sh` に rclone or AWS CLI で R2 アップロード追加
- 流れ: cron 日次 → `pg_dump` → gzip → batch コンテナ内から R2 アップロード → 7 日分保持
- 前提: Cloudflare R2 アカウント + バケット作成、API トークン取得

#### #27 batch cron 設定の詳細
- batch 日次 (JST 7:30): 市場データ取得 + 計算
- batch 週次 (金曜 JST 8:00): レイヤー再計算
- R2 バックアップ日次 (#34 と連動)
- ログ出力先: `/var/log/open-regime/batch.log` (logrotate 設定込み)
- エラー通知: Sentry (#35 と連動)

#### #33 ufw CF IP 制限の詳細
- Cloudflare の IP レンジ ([https://www.cloudflare.com/ips/](https://www.cloudflare.com/ips/)) を ufw に登録
- 80/443 を CF IP のみ許可、他は拒否
- 22/tcp は引き続き全許可 (SSH)
- CF が IP 追加した際にメンテが必要なので注意

#### #35 Sentry DSN 設定の詳細
- `.env` に `SENTRY_DSN` を追加 (api-python/api-go 共通)
- batch コンテナにも DSN 注入
- docker-compose.prod.yml の `SENTRY_DSN=${SENTRY_DSN:-}` は既に設定済み
