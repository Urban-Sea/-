### 2.2 なぜこの構成か

| 要件 | 解決策 |
|------|--------|
| 計算ロジックを隠したい | FastAPIをサーバー側に置く（フロントに露出しない） |
| Flaskを使いたくない | FastAPI（モダン、高速、型安全） |
| 複数人がアクセス | クラウドホスティング |
| 無料で運用 | Cloudflare Pages + Railway + Supabase（全て無料枠） |

## 3. データベース設計

### 3.1 設計方針

**新システムに持ち込むタブ:**
- 統合タブ
- 配管タブ
- 米国景気警戒タブ
- シグナルタブ
- 保有タブ

**保存するもの:**
- 銘柄マスター（カテゴリ、名前等）
- 流動性データ（FRBバランスシート、金利、クレジットスプレッド等）
- 経済指標（修正履歴対応・縦持ち方式で無限追加可能）
- 市場状態履歴（配管・統合タブ用）
- 個人データ（保有・取引・ウォッチリスト・設定）
- 一時キャッシュ（5分TTL、複数ユーザー共有）

**保存しないもの:**
- 株価OHLCV → yfinanceで都度取得
- リアルタイム価格 → yfinanceで都度取得
- EMA/ATR等 → 株価から都度計算
- シグナル結果 → Railwayで都度計算（DB保存不要）

### 3.4 テーブル定義（SQL）

```sql
-- =====================================================
-- 【シグナル分析用】
-- =====================================================

-- 銘柄マスター
CREATE TABLE stock_master (
    ticker VARCHAR(10) PRIMARY KEY,
    name VARCHAR(100),                  -- 'NVIDIA Corporation'
    sector VARCHAR(50),                 -- 'Technology'
    industry VARCHAR(100),              -- 'Semiconductors'

    -- カテゴリ
    price_category VARCHAR(20),         -- 'penny', 'mid', 'large'
    watchlist_category VARCHAR(50),     -- 'robotics', 'defense', 'ai'

    -- メタ情報
    market_cap BIGINT,
    exchange VARCHAR(10),               -- 'NASDAQ', 'NYSE'

    -- 状態
    is_active BOOLEAN DEFAULT TRUE,
    added_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 一時キャッシュ（5分TTL）
CREATE TABLE stock_cache (
    ticker VARCHAR(10) PRIMARY KEY,
    data JSONB,                         -- OHLCV + 分析結果
    fetched_at TIMESTAMPTZ DEFAULT NOW(),
    expires_at TIMESTAMPTZ DEFAULT NOW() + INTERVAL '5 minutes'
);

CREATE INDEX idx_stock_cache_expires ON stock_cache(expires_at);

-- =====================================================
-- 【配管タブ用 - 流動性グループ】
-- =====================================================

-- FRBバランスシート
CREATE TABLE fed_balance_sheet (
    date DATE PRIMARY KEY,
    reserves DECIMAL,                   -- 準備預金（10億ドル）
    rrp DECIMAL,                        -- リバースレポ（10億ドル）
    tga DECIMAL,                        -- 財務省当座預金（10億ドル）
    soma_assets DECIMAL,                -- SOMA資産合計（10億ドル）
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 金利
CREATE TABLE interest_rates (
    date DATE PRIMARY KEY,
    fed_funds DECIMAL,                  -- FFレート
    treasury_2y DECIMAL,                -- 2年国債利回り
    treasury_10y DECIMAL,               -- 10年国債利回り
    treasury_spread DECIMAL,            -- 10年-2年スプレッド
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- クレジットスプレッド
CREATE TABLE credit_spreads (
    date DATE PRIMARY KEY,
    hy_spread DECIMAL,                  -- ハイイールドスプレッド
    ig_spread DECIMAL,                  -- 投資適格スプレッド
    ted_spread DECIMAL,                 -- TEDスプレッド
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 市場指標
CREATE TABLE market_indicators (
    date DATE PRIMARY KEY,
    vix DECIMAL,                        -- VIX指数
    dxy DECIMAL,                        -- ドルインデックス
    sp500 DECIMAL,                      -- S&P500
    nasdaq DECIMAL,                     -- NASDAQ
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 銀行セクター（KRE）
CREATE TABLE bank_sector (
    date DATE PRIMARY KEY,
    kre_close DECIMAL,                  -- KRE終値
    kre_52w_high DECIMAL,               -- 52週高値
    kre_52w_low DECIMAL,                -- 52週安値
    kre_52w_change DECIMAL,             -- 52週変化率（%）
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- SRF利用額
CREATE TABLE srf_usage (
    date DATE PRIMARY KEY,
    amount DECIMAL,                     -- 利用額（10億ドル）
    source VARCHAR(20),                 -- 'manual' or 'h41'
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- 信用取引残高
CREATE TABLE margin_debt (
    date DATE PRIMARY KEY,
    debit_balance DECIMAL,              -- 信用取引残高（10億ドル）
    free_credit DECIMAL,                -- 無担保信用残高
    change_2y DECIMAL,                  -- 2年変化率（%）
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- MMF資産
CREATE TABLE mmf_assets (
    date DATE PRIMARY KEY,
    total_assets DECIMAL,               -- 総資産（10億ドル）
    change_3m DECIMAL,                  -- 3ヶ月変化率（%）
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- =====================================================
-- 【米国景気警戒タブ用 - 雇用グループ】
-- =====================================================

-- 経済指標マスター（修正履歴対応・縦持ち方式）
CREATE TABLE economic_indicators (
    id SERIAL PRIMARY KEY,
    indicator VARCHAR(50) NOT NULL,     -- 'NFP', 'GDP', 'CPI', 'UNEMPLOYMENT', etc.
    reference_period DATE NOT NULL,     -- 対象期間（2026-01-01 = 2026年1月分）
    current_value DECIMAL,              -- 最新の値
    revision_count INTEGER DEFAULT 0,   -- 修正回数

    -- 追加の雇用統計詳細（NFP用）
    nfp_change INTEGER,                 -- 前月比変化（千人）
    u3_rate DECIMAL,                    -- 公式失業率 U3（%）
    u6_rate DECIMAL,                    -- 実質失業率 U6（%）
    avg_hourly_earnings DECIMAL,        -- 平均時給（ドル）
    wage_mom DECIMAL,                   -- 平均時給 前月比（%）
    labor_force_participation DECIMAL,  -- 労働参加率（%）

    notes TEXT,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW(),

    UNIQUE(indicator, reference_period)
);

-- 経済指標の修正履歴（無限追加可能）
CREATE TABLE economic_indicator_revisions (
    id SERIAL PRIMARY KEY,
    indicator_id INTEGER REFERENCES economic_indicators(id) ON DELETE CASCADE,
    revision_number INTEGER NOT NULL,   -- 0=速報, 1=第1次改定, 2=第2次改定, ...
    value DECIMAL NOT NULL,
    published_date DATE NOT NULL,       -- 発表日
    change_from_prev DECIMAL,           -- 前回発表値からの変化
    change_pct_from_prev DECIMAL,       -- 前回発表値からの変化率（%）
    notes TEXT,
    created_at TIMESTAMPTZ DEFAULT NOW(),

    UNIQUE(indicator_id, revision_number)
);

CREATE INDEX idx_revisions_indicator ON economic_indicator_revisions(indicator_id);
CREATE INDEX idx_revisions_date ON economic_indicator_revisions(published_date);

-- 週次失業保険
CREATE TABLE weekly_claims (
    week_ending DATE PRIMARY KEY,       -- 週末日（YYYY-MM-DD）
    initial_claims INTEGER,             -- 新規失業保険申請件数
    continued_claims INTEGER,           -- 継続失業保険申請件数
    initial_claims_4w_avg INTEGER,      -- 新規申請4週移動平均
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- =====================================================
-- 【統合タブ用 - 履歴グループ】
-- =====================================================

-- マーケット状態履歴
CREATE TABLE market_state_history (
    id SERIAL PRIMARY KEY,
    date DATE NOT NULL UNIQUE,
    state VARCHAR(50) NOT NULL,         -- 'HEALTHY', 'MARKET_OVERSHOOT', etc.
    layer1_stress DECIMAL,
    layer2a_stress DECIMAL,
    layer2b_stress DECIMAL,
    credit_pressure VARCHAR(20),        -- 'Low', 'Medium', 'High'
    comment TEXT,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- レイヤーストレス履歴
CREATE TABLE layer_stress_history (
    id SERIAL PRIMARY KEY,
    date DATE NOT NULL,
    layer VARCHAR(20) NOT NULL,         -- 'layer1', 'layer2a', 'layer2b'
    stress_score DECIMAL NOT NULL,
    components JSONB,                   -- 詳細をJSON形式で保存
    created_at TIMESTAMPTZ DEFAULT NOW(),

    UNIQUE(date, layer)
);

-- =====================================================
-- 【保有タブ用 - 個人データ】
-- =====================================================

-- 保有銘柄
CREATE TABLE holdings (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE NOT NULL,

    ticker VARCHAR(10) NOT NULL,
    shares DECIMAL(12,4) NOT NULL,
    avg_price DECIMAL(12,4) NOT NULL,

    -- 取引情報
    entry_date DATE,
    account_type VARCHAR(20) DEFAULT 'tokutei',  -- 'nisa', 'tokutei'
    sector VARCHAR(50),                 -- 'Technology', 'Finance', etc.

    -- エントリー時の状態（V10用）
    regime_at_entry VARCHAR(20),        -- 'BULL', 'BEAR', 'RECOVERY', 'WEAKENING'
    rs_at_entry VARCHAR(20),            -- 'UP', 'FLAT', 'DOWN'
    fx_rate DECIMAL(8,4) DEFAULT 150.0, -- 為替レート

    -- 管理情報
    target_price DECIMAL(12,4),
    stop_loss DECIMAL(12,4),
    thesis TEXT,                        -- 投資理由
    notes TEXT,

    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

ALTER TABLE holdings ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Users can CRUD own holdings" ON holdings
    FOR ALL USING (auth.uid() = user_id);

-- 取引履歴
CREATE TABLE trades (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE NOT NULL,
    holding_id UUID REFERENCES holdings(id) ON DELETE SET NULL,

    ticker VARCHAR(10) NOT NULL,
    action VARCHAR(10) NOT NULL,        -- 'BUY', 'SELL'
    shares DECIMAL(12,4) NOT NULL,
    price DECIMAL(12,4) NOT NULL,
    fees DECIMAL(10,4) DEFAULT 0,

    trade_date TIMESTAMPTZ NOT NULL,
    account_type VARCHAR(20),

    -- エントリー/エグジット時の状態
    regime VARCHAR(20),                 -- 取引時のMarket Regime
    rs_trend VARCHAR(20),               -- 取引時のRS Trend

    -- 振り返り
    reason TEXT,                        -- entry_reason / exit_reason
    lessons_learned TEXT,

    -- 結果（SELL時）
    profit_loss DECIMAL(12,4),
    profit_loss_pct DECIMAL(8,4),
    holding_days INTEGER,

    created_at TIMESTAMPTZ DEFAULT NOW()
);

ALTER TABLE trades ENABLE ROW LEVEL SECURITY;

    FOR ALL USING (auth.uid() = user_id);

-- =====================================================
-- 現金・預金残高（2026-02-25追加）
-- =====================================================
CREATE TABLE cash_balances (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    user_id TEXT NOT NULL,

    label VARCHAR(100) NOT NULL,            -- 'SBI証券', '楽天証券' 等
    currency VARCHAR(10) NOT NULL DEFAULT 'JPY',  -- 'JPY', 'USD'
    amount DECIMAL(14,2) NOT NULL DEFAULT 0,
    account_type VARCHAR(20),               -- 'nisa', 'tokutei', 'bank'

    updated_at TIMESTAMPTZ DEFAULT NOW()
);

CREATE INDEX idx_cash_user ON cash_balances(user_id);

-- =====================================================
-- 日次ポートフォリオスナップショット（2026-02-25追加）
-- =====================================================
-- 毎日の終値でポートフォリオを時価評価し保存。
-- 資産推移チャートのデータソース。
-- バッチ日次処理（app/batch/snapshot.py）で自動生成。
CREATE TABLE portfolio_snapshots (
    id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
    user_id TEXT NOT NULL,
    snapshot_date DATE NOT NULL,

    total_market_value_usd DECIMAL NOT NULL,  -- Σ(shares × close_price)
    total_cost_usd DECIMAL NOT NULL,          -- Σ(shares × avg_price)
    unrealized_pnl_usd DECIMAL NOT NULL,      -- market_value - cost
    cash_usd DECIMAL NOT NULL DEFAULT 0,      -- JPY現金をUSD換算 + USD現金
    total_assets_usd DECIMAL NOT NULL DEFAULT 0, -- market_value + cash
    fx_rate_usdjpy DECIMAL,                   -- 換算に使ったレート
    holdings_count INTEGER NOT NULL DEFAULT 0,
    holdings_detail JSONB,                    -- 銘柄別詳細（ticker, shares, close_price等）

    created_at TIMESTAMPTZ DEFAULT NOW(),
    CONSTRAINT uq_snapshot UNIQUE (user_id, snapshot_date)
);

CREATE INDEX idx_snapshots_user_date
    ON portfolio_snapshots (user_id, snapshot_date DESC);

-- =====================================================
-- updated_at 自動更新トリガー
-- =====================================================
CREATE OR REPLACE FUNCTION update_updated_at()
RETURNS TRIGGER AS $$
BEGIN
    NEW.updated_at = NOW();
    RETURN NEW;
END;
$$ LANGUAGE plpgsql;

CREATE TRIGGER stock_master_updated_at
    BEFORE UPDATE ON stock_master
    FOR EACH ROW EXECUTE FUNCTION update_updated_at();
```

## 4. キャッシュ戦略

### 4.1 なぜキャッシュが必要か

```
問題: yfinanceにはレート制限がある

ユーザーA → NVDAを見る → yfinance API
ユーザーB → NVDAを見る → yfinance API  ← 無駄！

解決: 一時キャッシュ（5分TTL）

ユーザーA → NVDAを見る → yfinance API → DBに一時保存
ユーザーB → NVDAを見る → DBから返す（API呼ばない）
5分後 → 自動削除
```

### 4.2 キャッシュのライフサイクル

```
1. ユーザーが銘柄を要求
2. DBのstock_cacheを確認
   ├── あり＆有効期限内 → DBから返す
   └── なし or 期限切れ → yfinance API呼び出し → DBに保存 → 返す
3. 定期的に期限切れを削除（cron or GitHub Actions）
```

### 4.3 キャッシュクリーンアップ

```sql
-- 5分ごとに実行
DELETE FROM stock_cache WHERE expires_at < NOW();
```

## 15. 日次ポートフォリオスナップショット（2026-02-25実施）

### 15.1 概要

保有銘柄の時価を毎日の終値で評価し、`portfolio_snapshots` テーブルに保存するシステム。
資産推移タブのデータソースとなる。

**なぜ必要か:**
- 取引履歴だけでは「取得原価」しかわからない
- 株価の日々の変動（含み損益の推移）を追跡するには、毎日のスナップショットが必須

### 15.2 データフロー

```
日次バッチ (_run_daily)
  ├── FRED/Yahoo/NY Fed データ取得 → Supabase upsert
  ├── fetch_market_indicators → upsert （USD/JPY がここで保存）
  ├── fetch_bank_sector → upsert
  └── take_daily_snapshot()  ← ここで実行
        ├── holdings テーブル → 全ユーザーの保有銘柄取得
        ├── yf.download(全ティッカー, period="5d") → 終値取得（1回のAPI呼び出し）
        ├── market_indicators → USD/JPY レート取得
        ├── cash_balances → 全ユーザーの現金残高取得
        └── ユーザーごとにスナップショット行構築 → portfolio_snapshots に upsert
```

### 15.3 スナップショット行の内容

| フィールド | 説明 |
|-----------|------|
| `total_market_value_usd` | Σ(shares × close_price) — 時価評価額 |
| `total_cost_usd` | Σ(shares × avg_price) — 取得原価 |
| `unrealized_pnl_usd` | market_value - cost — 含み損益 |
| `cash_usd` | USD現金 + JPY現金/fx_rate — 現金合計(USD換算) |
| `total_assets_usd` | market_value + cash — 総資産 |
| `fx_rate_usdjpy` | 換算に使ったレート |
| `holdings_detail` | 銘柄別詳細JSONB（ticker, shares, close_price, market_value等） |

### 15.4 バッチモジュール

| ファイル | 役割 |
|---------|------|
| `app/batch/snapshot.py` | 日次スナップショット（`take_daily_snapshot()`） |
| `app/batch/backfill_snapshots.py` | 過去データバックフィル（取引履歴から再構築） |
| `app/batch/db.py` | `upsert_portfolio_snapshots()` |
| `app/batch/run.py` | `--snapshot` / `--backfill-snapshots` フラグ |

### 15.5 過去データバックフィル

取引履歴から過去のポートフォリオ状態を日ごとに再構築:

```
python app/batch/run.py --backfill-snapshots --since 2024-06-01 --verbose
```

1. 全ユーザーの trades を trade_date ASC で取得
2. 日付ごとに holdings_state を更新（BUY: 追加, SELL: 減少）
3. `yf.download(all_tickers, start=since, end=today)` で過去終値を一括取得
4. 営業日ごとにスナップショット行を生成 → upsert

**注意:** バックフィル分は `cash_usd=0` （過去の現金推移は不明のため）

### 15.6 APIエンドポイント

```
GET /api/holdings/portfolio-history?months=24
```

`portfolio_snapshots` テーブルからユーザーの日次スナップショットを返却。
レスポンス:
```json
{
  "history": [
    {
      "date": "2025-01-15",
```

## 17. DB容量管理・スケーリング戦略

### 17.1 現在の容量見積もり

| テーブル | 1行あたり | 行数/日 | 年間行数 | 年間容量 |
|---------|----------|---------|---------|---------|
| portfolio_snapshots | ~1KB (JSONB含む) | ユーザー数 | ~365×N | ~365KB×N |
| market_indicators | ~100B | 1 | ~250 | ~25KB |
| economic_indicators | ~100B | ~20 | ~5,000 | ~500KB |
| fed_balance_sheet | ~80B | 1 | ~250 | ~20KB |

**ユーザー5人の場合:** portfolio_snapshots = 年間 ~1.8MB
**Supabase無料枠:** 500MB（累計、年リセットなし）

### 17.2 スケーリング戦略

**Phase 1（現在〜500MB以内）:**
- 日次スナップショット（毎営業日）
- 何も変更不要

**Phase 2（容量が増えてきたら）:**
- スナップショット頻度を**2週間に1回**に変更
  - `app/batch/run.py` の `take_daily_snapshot()` 呼び出しに曜日条件を追加
  - 例: `if datetime.now().weekday() == 4:` (毎週金曜のみ)
- 古いスナップショットを間引き（月初のみ保持）
  - `DELETE FROM portfolio_snapshots WHERE snapshot_date < '2024-01-01' AND EXTRACT(DAY FROM snapshot_date) != 1`

**Phase 3（500MB超え/有料化検討時）:**
- Supabase Pro ($25/月, 8GB)
- または holdings_detail JSONB を別テーブルに分離（本体テーブルを軽量化）
- パーティショニング（年別テーブル分割）

### 17.3 容量モニタリング

Supabase ダッシュボード → Database → Disk Usage で確認可能。
300MBを超えたら間引き戦略の導入を検討。
